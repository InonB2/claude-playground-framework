### WhatsApp Bridge ‚Äî UNBLOCKED ‚úì
**Root cause identified:** Windows security (`WinDefend` + Windows internals) blocks Go's `go tool buildid -w` step, which writes a build ID into a freshly-linked `.exe` right after the linker creates it. The file is momentarily locked by the security scanner at the instant `buildid` tries to write.

**Fix applied (permanent, no AV disabling needed):**
1. Replaced `github.com/mattn/go-sqlite3` (CGO-required) with `modernc.org/sqlite` (pure-Go, no C compiler needed) ‚Äî avoids CGO toolchain entirely
2. Added `sqlite3_compat.go` shim that registers `modernc.org/sqlite` as the `"sqlite3"` driver name (whatsmeow uses `"sqlite3"` by default)
3. Build flags: `-ldflags="-buildid="` + `CGO_ENABLED=0` + `GOTMPDIR=D:\go-temp` ‚Äî skips the `buildid -w` step entirely
4. Created `C:\tools\whatsapp-mcp\whatsapp-bridge\rebuild.ps1` for future rebuilds (no manual flag handling)

**Result:** `whatsapp-bridge.exe` (27MB) built, tested, and confirmed running:
- Starts cleanly, no SQLite error
- Database auto-migrated (v1‚Üív6)
- Bridge process runs (PID confirmed)
