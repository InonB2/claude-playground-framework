# Session Log — 2026-05-10 (Evening)

## What Was Accomplished

**Telegram MCP server — .env CRLF bug fixed.**
`plugin:telegram:telegram` was showing "Failed to connect" despite the bot token being written to `~/.claude/channels/telegram/.env` in the previous session. Root cause isolated: the `.env` was saved with Windows `\r\n` line endings. The `server.ts` parser splits on `\n` then runs `/^(\w+)=(.*)$/` — the trailing `\r` caused every line to fail the regex, so `TELEGRAM_BOT_TOKEN` was never loaded into `process.env`. Fixed by rewriting the file with LF-only line endings via `[System.IO.File]::WriteAllText(...)`. Server now starts cleanly and loads the token. Verified with a bun debug script and a live server test.
